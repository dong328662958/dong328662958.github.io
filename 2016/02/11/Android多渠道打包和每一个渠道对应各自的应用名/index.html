Android多渠道打包和每一个渠道对应各自的应用名

发表于 2016-02-11   |   分类于 Android   |  

以下基于友盟多渠道打包为例:

在application中配置meta-data

配置项目中的 build.gradle文件:

android {       
   //签名信息. app根目录下,与libs同级
    signingConfigs {
        myConfig {
            storeFile file("appkey.keystore")
            storePassword "===="
            keyAlias "===="
            keyPassword "===="
        }
    }


    flavors()//调用flavors方法

    buildTypes {
        release {
            signingConfig signingConfigs.myConfig
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguard-rules.pro'

            //自定义生成的apk名称,默认为工程名
            //如果不需要,可以注释掉
            //以下,输出为 dfy1.0_baidu.apk
            applicationVariants.all { variant ->
                variant.outputs.each { output ->
                    def outputFile = output.outputFile
                    if (outputFile != null && outputFile.name.endsWith('.apk')) {
                        def fileName = "dfy${defaultConfig.versionName}_${variant.productFlavors[0].name}.apk"
                        output.outputFile = new File(outputFile.parent, fileName)
                    }
                }
            }

      }
    }

}

定义flavors方法,该方法与android{}同级
    def flavors(){
        def path = "./channels.txt"
        file(path).eachLine {
            line ->
                if (!line.startsWith("//")){
                    android.productFlavors.create(line,{
                        manifestPlaceholders = [UMENG_CHANNEL_VALUE: name]
                    })
                }
        }
    }

在工程跟目录下,与src同级,创建 channels.txt,写入channel值,注意前面不能留空格.
